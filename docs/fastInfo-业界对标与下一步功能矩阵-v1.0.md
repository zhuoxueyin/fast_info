## 0. fastInfo 定位复述（2026-07-04 早上 7:50 用户原话）

> "它核心是通过 AI 来驱动这个资讯情报的。
> 传统的资讯，比如微博或头条，是由人主动推荐，或者人自己去浏览。
> 而我们的产品支持用户自定义的订阅，用户可以通过对话告诉系统自己想订阅、想看什么。
> 比如，我今天想关注一下世界杯，系统就会根据我们整体的资讯池，为我订阅世界杯足球相关的内容。"

**一句话轴**:**AI 驱动 + 用户 NL 对话式订阅 + 统一资讯池 + 临时场景(世界杯例子)**。

## 1. 业界对标矩阵

> ⛳ 标尺:**AI 驱动** / **NL 创建订阅** / **用户主动权** / **临时兴趣场景** / **统一资讯池**

### 1.1 国际 4 家

| 产品 | 类别 | AI 驱动 | NL 创建订阅 | 用户主动权 | 临时场景 | 统一资讯池 | 总评 |
|---|---|:-:|:-:|:-:|:-:|:-:|---|
| **Feedly AI**(Leo Pro) | RSS 工具进化版 | ✅ 部分 | ❌ 仍要手选源 | ❌ | ❌ | ❌ | 行业风向被引,2026-05 总榜涨 38 位。但订阅模型仍是「选源 + 关键词」古典模式。 |
| **Perplexity Discover** | 搜索衍生 feed | ✅ | ⚠️ 半 NL（query 当 topic) | ❌ | ⚠️ | ❌ | 没有"订阅"概念,query 即 topic。 |
| **Spotify AI DJ** | 音频 | ✅ | ✅ "我今天想听轻爵士" | ⚠️ | ⚠️ | ❌ | 范式参考价值最大,但音频领域。 |
| **Perplexity Personal Computer**(2026-04-17) | 桌面 AI | ✅ | ⚠️ | ✅ | ❌ | ❌ | 把 AI 入口从搜扩到桌面调度,佐证"AI 入口下沉"是趋势。 |

### 1.2 国内 2 家 + 范式参考

| 产品 | 类别 | 评 |
|---|---|---|
| **idcos.com「RSS 订阅管理 + AI 日报」** | 极简 RSS aggregator | 形态**最接近 fastInfo MVP5**,但只做"每天 10 点聚合报告",**没 NL 创建订阅**、**没 KOL 抓取**、**没多渠道推送**、**没推荐**。差异化空间大。 |
| **即刻 / 小红书 discovery / 得到** | 算法 / 编辑驱动 | 都是"平台推给用户",主动权在平台,而**不在用户**。fastInfo 完全反向。 |
| **(历史) Readhub** | AI 资讯聚合(已逝 2018) | 范式参考:专业编辑 + 技术整合,死因:窄、小众、商业化弱。说明这条赛道要活下来必须加 AI + 个性化。 |

### 1.3 OSS / 开发中 3 家

| 项目 | 评 |
|---|---|
| `stevenbaert/AI-NewsAggregator` | 极简 AI + 邮件推送(单文件级) |
| `geophyzik/rss-aggregator` | 老式 RSS 工具 |
| `iamgarima/news-aggregator` | 教学级 GitHub release |

> **结论**:**业界现在没有"AI 驱动 + NL 订阅 + 临时场景"三者全打的玩家**。fastInfo 定位抢在行业前沿(Feedly 在向 AI 转,但慢),窗口期约 6-12 个月。

## 2. 下一步功能矩阵（按维度切，按 P0/P1/P2 排）

### 🟢 P0 · 1 周内必须上（直接体现"AI 驱动 + NL 订阅"差异化）

| 维度 | 功能 | 描述 |
|---|---|---|
| **🆕 临时兴趣(核心场景)** | **"今日话题"** 模式 | 用户 NL "今天关注世界杯" → **不建 subscription**,只生成一张**临时 dashboard**(24h 失效),所有符合的内容进这个 workspace。点击"持续关注"才转 subscription。**这就是用户给的例子** |
| **🆕 NL 改订阅** | 对话式 PATCH | `/api/subs/{id}/nl-patch` 接"把这个订阅改成每天下午 5 点,只发到飞书"。NL→PATCH,比手填 cron 友好 10 倍 |
| **🆕 AI 推荐主题卡** | HomePage 顶部"你可能想订阅" | 基于用户历史订阅 + items pool 热点 → LLM 推荐 3-5 个主题卡,一键订阅 |

### 🟡 P1 · 1 个月内（AI 增值 + 体验提升）

| 维度 | 功能 |
|---|---|
| 🃏 **推荐卡** | "摘要 + 收藏 + 关注作者"三件套,点击收藏即进用户收藏夹;点击关注作者进 auto-subscribe |
| 📰 **AI 编辑视角** | "用 3 句话帮我讲清楚这篇",点击文章进 deep_interpretation 模式 |
| 💬 **追问 / 对话式深读** | 进 ItemDetail 后,底部输入框,AI 跟用户对话式解释这篇文章 |
| 📥 **个性化解读(已架构支持)** | push 时根据用户阅读历史动态调整 max_tokens/关键点 |
| 🔍 **检索 v2(DashScope Embedding + LanceDB)** | 替代 MongoDB text 兜底,中文精确 |

### 🔵 P2 · 季度级（平台化 / 协作）

| 维度 | 功能 |
|---|---|
| 👥 **订阅共享 / 协作** | 用户 A 订阅"AI 资讯"分享给用户 B 一键订阅 |
| 🏷️ **订阅市场** | 其他用户的高质订阅按热度 + 分类页展示,一键套用 |
| 🎬 **多模态**(视频 / 音频 / 图片摘要) | X v2 真实 API 接入 + 视频流摘要 |
| 📱 **iOS / Android Native**(PWA → Native) | 当前是 PWA + /m 响应式,可升级 |
